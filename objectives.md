
# Objectives

1. I want to build out this repository as a resource for helping me build competitive pokemon teams. To make this repository an ever-growing resource that can remain current during the ever-changing competitive meta game, I want to create Python utilities that can go out to online resources to gather current information on the most prominent teams and statistical overviews of whatever the current competitive format is and save those utilities in the tools directory. I hope to collect snapshots of individual research sessions as annotated bibliographic reference documents saved to the docs directory for future reference and to create a memory for future revisits to this project.

2. To start, I want to begin by simply researching what common community references exist for competitive pokemon team building and meta game analysis and tag them as persistent locations to reference either by a Claude search agent or future Python utilities. This can be constructed as a build out of an annotated bibliography for common research resources that can be saved to the docs directory.

3. Right now I am aiming to build a team in the brand new release, Pokemon Champions, which is a free to play game focused entirely on competitive pokemon battling. I want to create reference documentation for what competitive pokemon battling is in general as a reference for Claude agents to read as they explore this repository to guide the decision making process in a more comprehensive way. I myself am familiar with the mechanics of the game and have played competitive pokemon before in the Generation 9 games Scarlet and Violet, reaching Master Ball rank during a single-restricted format using a Calyrex Shadow Rider focused team, however I believe that having a comprehensive overview will be helpful for Claude agents that are leveraging the contents of this repository to assist in team building. Team building as a concept is broadly applicable to any game or format, however there are distinct differences that will likely need to be catalogued into reference documentation to ensure that the advice provided during a particular format is in alignment with the ruleset and distinctions between rule sets and games.

4. It may be useful to slowly build out information about different common pokemon, their entire learnset and abilities and common items, and common teammates, as a springboard for team building based on statistical analysis of the meta game at any point. This could be done with a Python playwright bot that can acquire pokemon data from common public resources, but I want to save this data in a way that will be easily referenced by Claude agents when thinking through what specific team to build around a particular concept.

5. As a first experiment, I want to build a team around Mega Dragonite, as Dragonite is one of my favorite pokemon. This can be a first test case of the usefulness of the repository of information that I am trying to collect here.
